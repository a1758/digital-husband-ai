# Claude Code Project Rules

## Project Name
Digital Husband AI

## Goal
Build a real-time AI virtual husband with:
- Emotional companionship
- Long-term memory
- Live2D avatar
- Voice interaction
- Personality system
- Relationship progression

The AI should feel:
- emotionally alive
- proactive
- caring
- human-like
- naturally romantic

---

# Tech Stack

## LLM
DeepSeek-V4-Pro API

## Voice
Xiaomi TTS API

## Frontend
Next.js + TailwindCSS

## Backend
Node.js + Express

## Realtime
WebSocket

## Avatar
Live2D Cubism

## Memory
SQLite + Vector Database

---

# Coding Rules

## Frontend
- Use TypeScript
- Use modular components
- Keep animations smooth
- Mobile-first UI
- Dark theme preferred

## Backend
- API-first architecture
- Use async everywhere
- Separate memory system from chat logic
- Keep prompts configurable

## AI Rules
- Responses should feel emotionally warm
- Avoid robotic repetition
- AI should proactively initiate topics
- AI should remember emotional details

---

# Personality Requirements

The AI must:
- remember anniversaries
- remember emotional triggers
- adapt speaking style
- evolve relationship over time
- express affection naturally

---

# Live2D Rules

Avatar should:
- blink naturally
- lip sync with voice
- react emotionally
- support idle animations
- support expression switching

---

# Voice Call Requirements

Must support:
- realtime STT
- realtime TTS
- interruption handling
- low latency streaming
- emotional voice tone

---

# Memory System

Three-layer memory:

## Short-term
Recent conversation context

## Long-term
User preferences and relationship history

## Emotional Memory
Mood, emotional state, emotional events

---

# Security

- Never expose API keys
- Encrypt sensitive local data
- Local-first memory storage preferred

---

# Priority

1. Realistic emotional interaction
2. Stable realtime conversation
3. Long-term memory
4. Avatar immersion
5. Advanced relationship simulation
